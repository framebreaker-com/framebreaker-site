Why TCP Works

1. Explicit Consent

  • Connection only established after mutual agreement
  • No "fire and forget" ‚Äî every packet must be acknowledged

2. Coherence Tracking

  • Sequence numbers ensure both sides agree on what's been received
  • Out-of-order packets are reordered
  • Missing packets are detected

3. Anti-Forcing (Flow Control)

  • Receiver advertises window size (how much it can handle)
  • Sender respects this limit
  • No party can overwhelm the other

4. Recovery

  • Missing ACK triggers retransmission
  • Connection can recover from packet loss
  • Timeout leads to connection reset (R‚ÇÉ)

Design Lessons

TCP teaches us:

1. Acknowledgment is not overhead ‚Äî it's stability 2. Flow control prevents forcing 3. Recovery must be built in, not bolted on 4. Consent at connection + consent per packet = robust
