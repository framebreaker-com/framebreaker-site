UAP in Privacy Law

Making GDPR and ePrivacy Actually Work

The Problem

Privacy laws like GDPR already require:

  • Freely given consent
  • Specific, informed consent
  • Unambiguous consent
  • Easy withdrawal
In practice, we get:
  • Cookie walls
  • Pre-ticked boxes
  • 47-page privacy policies
  • "By continuing, you agree..."
The law has the right words. The implementation has the wrong grammar.

Implementation Checklist

For Consent

  • [ ] Is consent explicit (not implied)?
  • [ ] Is "no" as easy as "yes"?
  • [ ] Is understanding verified?
  • [ ] Can consent be easily withdrawn?
  • [ ] Is each purpose separately consented?

For Transparency

  • [ ] Is information in plain language?
  • [ ] Is understanding verified (not assumed)?
  • [ ] Can users ask clarifying questions?
  • [ ] Are changes communicated and re-consented?

For Rights

  • [ ] Is the request process simple?
  • [ ] Is understanding confirmed before action?
  • [ ] Is the result verified with the user?
  • [ ] Is recovery available if something went wrong?
