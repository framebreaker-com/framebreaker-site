UAP Protocol Template

Design Guide for UAP-Conformant Protocols

Overview

This document provides a template for designing new protocols that conform to UAP principles.

A UAP-conformant protocol is more likely to:

  • Achieve stable operation
  • Resist common attack vectors
  • Recover gracefully from failures
  • Scale without losing coherence

Anti-Patterns

1. "Implicit Consent"

// BAD: Assuming consent
if (request_received) {
    process(request);  // No acknowledgment!
}

// GOOD: Explicit consent gate
if (request_received && acknowledged(request)) {
    process(request);
}

2. "Fire and Forget"

// BAD: No verification
send(data);

// GOOD: Verify coherence
send(data);
ack = await_acknowledgment();
if (!ack) trigger_recovery();

3. "Unlimited Power"

// BAD: No forcing prevention
execute_all_permissions(user);

// GOOD: Scoped permissions
execute_within_scope(user, granted_scope);

Final Checklist

Before deploying your protocol:

  • [ ] Can I explain the consent gate in one sentence?
  • [ ] What happens if acknowledgment never arrives?
  • [ ] How does a malicious party try to force state changes?

If you can't answer these, the protocol isn't ready.
