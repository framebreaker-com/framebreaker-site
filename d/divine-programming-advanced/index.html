II. The MUST/MAY/MUST-NOT Matrix

Classification

TypeMeaningConsequence of Violation
MUSTRequiredBuild fails
MAYOptionalBuild continues
MUST-NOTForbiddenBuild fails if present

Example

MUST:     ConsentGate maintains IMPLICIT = ABSENT
MAY:      Additional logging for debug
MUST-NOT: Hardcoded API keys in source

Why This Matters

Without explicit MUST/MAY/MUST-NOT:

  • AI interprets freely
  • Scope creeps
  • Governance leaks
With explicit matrix:
  • Constraints are testable
  • Violations are detectable
  • Builds are reproducible

VII. Labels for Epistemic Hygiene

The Four Labels

LabelMeaning
[FACT]Verified, observable, testable
[INFERENCE]Derived from facts, but not directly observed
[HYPOTHESIS]Proposed explanation, not yet tested
[PROPOSED]Suggested but not committed

Why Label Everything

Without labels: claims blend into assumptions. With labels: every statement has a testability rating.
